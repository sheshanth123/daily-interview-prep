## **Part 1: LLM Fundamentals**

### **1. What is an LLM and how does "next-token prediction" work?**
A Large Language Model (LLM) is a type of neural network based on the Transformer architecture, trained on massive datasets to understand and generate human-like text.

**Next-token prediction** is the core objective. Given a sequence of tokens (words or sub-words), the model calculates a probability distribution over the entire vocabulary to determine which token is most likely to follow.

*   **Example:** For the input "The capital of France is", the model assigns high probability to "Paris" and low probability to "Apple."
*   **The Math:** If $z$ represents the output logits from the final layer, the probability $P$ of the next token $i$ is calculated using the Softmax function:
$$P_i = \frac{e^{z_i}}{\sum_{j} e^{z_j}}$$

### **2. Pretraining vs. Fine-tuning vs. Instruction Tuning**
*   **Pretraining:** Training from scratch on a massive, unlabeled corpus (e.g., Wikipedia, Common Crawl). The model learns grammar and general knowledge.
*   **Fine-tuning:** Taking a pretrained model and training it further on a smaller, specific dataset (e.g., medical records) to adapt it to a domain.
*   **Instruction Tuning:** A subset of fine-tuning where the model is trained on "Prompt-Completion" pairs to follow specific commands (e.g., "Summarize this...").

### **3. Why do LLMs hallucinate?**
Hallucination occurs when a model generates factually incorrect but confident-sounding text.
*   **Causes:** Training data noise, "stochastic parroting" (predicting the most likely word even if false), or context window limitations.
*   **Reduction:** Use **RAG** (external grounding), **Chain of Thought (CoT)** prompting, or adjusting **Temperature** settings.

### **4. How does Temperature affect output?**
Temperature ($T$) scales the logits before the Softmax layer.
*   **Low $T$ (e.g., 0.1):** Makes the distribution "sharper." The model becomes deterministic, choosing only the most likely tokens.
*   **High $T$ (e.g., 0.8):** Flattens the distribution. The model becomes more creative/random.

$$P_i = \frac{\exp(z_i / T)}{\sum \exp(z_j / T)}$$

## **Part 2: Retrieval-Augmented Generation (RAG)**



### **1. Why RAG instead of Fine-tuning?**
| Feature | RAG | Fine-Tuning |
| :--- | :--- | :--- |
| **Knowledge** | External/Dynamic (updated easily) | Internal/Static (requires retraining) |
| **Transparency** | High (provides citations) | Low (Black box) |
| **Cost** | Lower (Infra for Vector DB) | Higher (GPU training time) |
| **Use Case** | Fact-heavy tasks | Style/Format/Tone adaptation |

### **2. Chunking Strategies and Overlap**
*   **Chunking:** Breaking large documents into smaller pieces (e.g., 500 tokens) because LLMs have limited context windows and retrieval is more precise with smaller segments.
*   **Overlap:** Including a small portion of the previous chunk (e.g., 10%) in the current chunk.
    *   **Why?** To ensure semantic context isn't lost if a crucial sentence is split between two chunks.

### **3. How to evaluate retrieval quality?**
*   **Precision@K:** Proportion of relevant documents in the top $K$ retrieved results.
*   **MRR (Mean Reciprocal Rank):** Evaluates where the first relevant document appears.
$$MRR = \frac{1}{|Q|} \sum_{i=1}^{|Q|} \frac{1}{rank_i}$$
*   **RAGAS Metrics:** Specifically for LLMs—**Faithfulness** (is the answer derived from the context?) and **Answer Relevance** (does it answer the prompt?).

## **Part 3: Machine Learning & Optimization**

### **1. How do you handle imbalanced datasets?**
*   **Resampling:** Oversampling the minority class (SMOTE) or undersampling the majority class.
*   **Cost-sensitive learning:** Assigning higher weights to the minority class in the loss function.
*   **Metrics:** Using F1-Score or Precision-Recall curves instead of Accuracy.

### **2. Parameter Selection: Top-k vs. Top-p**
*   **Top-k:** The model picks from the top $k$ most likely next tokens.
*   **Top-p (Nucleus Sampling):** The model picks from the smallest set of tokens whose cumulative probability exceeds threshold $p$. This is more dynamic than Top-k.

**Coding Example (HuggingFace Transformers):**
```python
outputs = model.generate(
    input_ids, 
    do_sample=True, 
    top_k=50, 
    top_p=0.95, 
    temperature=0.7
)
```

## **Part 4: Transformer Architecture**



### **1. Why is Attention better than Recurrence (RNN/LSTM)?**
*   **Parallelization:** RNNs process tokens sequentially (one by one), making them slow. Transformers process the whole sequence at once.
*   **Long-range Dependencies:** RNNs forget early information (vanishing gradient). The **Self-Attention** mechanism allows every token to "look at" every other token regardless of distance.

### **2. Self-Attention Complexity**
The complexity of standard self-attention is **$O(n^2 \cdot d)$**, where $n$ is sequence length and $d$ is embedding dimension. This is why long context windows (like 128k+) are computationally expensive—doubling the text quadruples the memory/compute needed.

## **Part 5: Advanced & Domain Specific**

### **1. What is an "Opus" model?**
"Opus" refers to **Claude 3 Opus**, the most powerful model in Anthropic's Claude 3 family. It is designed for complex reasoning, coding, and high-level strategy. It represents the "heavyweight" tier, compared to "Sonnet" (balanced) and "Haiku" (fast/cheap).

### **2. Multi-Agent Systems**
Instead of one LLM doing everything, you use multiple specialized agents (e.g., using **CrewAI** or **LangGraph**).
*   **Example:** Agent A searches the web, Agent B writes the code, and Agent C reviews the code.
*   **Benefit:** Better error handling and higher quality for complex workflows.
